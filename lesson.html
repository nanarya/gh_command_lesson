レッスン概要

進行方針
一番進捗・理解が遅い人に合わせて進行します。わからないことがあれば遠慮なく聞いてください。

このレッスンでは、Claude Code を使って GitHub CLI（gh コマンド）のインストールから実際の操作までを体験します。

事前準備
作業ディレクトリの作成など、Claude Code を始める前の環境整備
GitHub連携
gh コマンドのインストール・認証・基本操作の習得
実践操作
クローン・issue作成など実際の GitHub 操作を体験

事前準備（Claude Code 外での作業）

Claude Code を起動する前に、作業用ディレクトリを用意しておきましょう。

作業用ディレクトリの作成

Claude にリポジトリに紐づかない雑多な作業をさせるためのディレクトリを作成します。今後も継続して使うディレクトリです。

推奨パス
~/claude-workspace（名前は何でもOK）
用途
リポジトリに紐づかない雑多な作業をClaudeにお願いするときのディレクトリ
保持
今後も残しておいて継続利用する

GitHub連携（Claude Code 内での作業）

前の手順で作った作業用ディレクトリで Claude Code を起動し、以下の操作を行います。

gh コマンドのインストールと設定

Claude に次のプロンプトを順番に入力して、gh コマンドをインストールします。

「windowsでghコマンドをインストールする方法を教えて」
「その手順を実行して」
「インストール後の認証の手順を教えて」

gh コマンドでできることを理解する

gh コマンドと git コマンドの違いを把握しましょう。

「ghコマンドでできることを教えて」
「gitコマンドとの違いを教えて」
コマンド 主な役割 操作対象
git ローカルのバージョン管理 ローカルリポジトリ・コミット・ブランチ
gh GitHub API へのアクセス Issue・PR・Release・GitHub の設定など

gh コマンドの動作確認

「ghコマンドが使えるか確認してみて」

リポジトリをクローンしてみよう

まず ~/repos ディレクトリを作ります。今後リポジトリをクローンするときに使うディレクトリです。

「このリポジトリをghコマンドで~/repos/にクローンして。 git@github.com:nanarya/gh_command_lesson.git」

リポジトリ内での操作

クローンしたリポジトリ（~/repos/gh_command_lesson）に Claude Code を移動して作業します。

gh コマンドで気をつけること

注意点
gh コマンドや git コマンドを使うと、リポジトリ内のコードの変更ができてしまいます。必要に応じて読み取り専用に制限しましょう。

おすすめ設定：読み取り専用にする

「このリポジトリ内でghコマンドおよびgitコマンドでコードの変更はできないようにして、読み取りのみできるようにして」

※ 例として先に settings.json をコミットしておきたかったので、このプロンプトはすでに実行してリポジトリにコミット済みです。コミット済みファイルは適当に作って精査していないので信頼しすぎないでください。

作業ディレクトリの使い方を確認

どこで Claude を動かすかについて、使い方が固まっているか確認しましょう。Claude Code を起動するディレクトリには3種類の使い分けがあります。

ディレクトリ 用途
~/claude-workspace Claude の雑多な作業ディレクトリ。リポジトリに紐づかない作業をさせる場所
~/repos/ リポジトリのクローン先。コードを伴う作業の主な作業場
OneDrive / SharePoint のフォルダ ドキュメント系ファイルの置き場。Claude Code をここで起動してドキュメント作業をさせる
